# Decisões Técnicas

> Este arquivo registra as principais decisões técnicas tomadas durante o desenvolvimento da Sprint 06. Cada decisão segue o formato ADR (Architecture Decision Record) simplificado.

---

## ADR-001 · Escolha da linguagem: Python

**Data**: 2026-05-27
**Status**: Aceito

**Contexto**: a PoC envolve manipulação de dados, possíveis modelos preditivos e simulação rápida de cenários.

**Decisão**: usar Python como linguagem principal.

**Justificativa**: a equipe já trabalhou com Python na Sprint 04 (análise de dados), o ecossistema (pandas, numpy, matplotlib, scikit-learn) cobre todas as necessidades previstas e o tempo de desenvolvimento é curto.

---

## ADR-002 · Kanban no GitHub Projects

**Data**: 2026-05-27
**Status**: Aceito

**Contexto**: precisamos de um Kanban público e integrado ao código.

**Decisão**: usar GitHub Projects (board nativo do repositório).

**Justificativa**: integra com issues e pull requests, é gratuito, público por padrão quando o repo é público, e dispensa criar conta em ferramenta externa.
